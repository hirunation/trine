# Talent

The builders this project seeks think in constraints, not features. They find satisfaction in removing code rather than adding it, in proving a bound rather than shipping a flag, in making the system smaller rather than more configurable. If you read the 270-byte ROM derivation and your reaction is "why isn't it 200," you belong here.

Precision is non-negotiable. Every function has a contract. Every benchmark has a reproduction script. Every claim in the documentation is backed by a test that will fail if the claim becomes false. Tolerance for ambiguity is low. Tolerance for rigor is infinite.
